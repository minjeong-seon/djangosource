Process 요구사항 정리

※목적: 메일 발송 자동화 기능, 랜덤박스 프로세스 자동화 기능, Ajax로 비동기식으로 기능 구현해보기


0. 회원가입 시:
    > 가입정보 - 이메일, 주소 추가
    > 주소 입력 시 - 다음(카카오) 주소 API 활용


1. 사용자가 구매버튼 클릭 시:
	> 해당 유저 구매금액 업데이트

	> 남은 매출액이나 총 매출액이 브랜드 최저가 이하인 경우:
		> 구매 수량만큼 일반 상품 랜덤 증정
		> 일반 상품 재고 감소

	> 총 매출액이 브랜드 상품 최저가액 보다 크거나 같은 경우:
		> 해당 브랜드 상품 재고 감소
		> 랜덤 유저에게 해당 상품 증정
		> 해당 유저에 당첨 메일 발송
        > 단, 이미 구매 수량만큼 일반 상품에 당첨된 유저는 당첨 제외


2. 관리자가 QnA 게시판 답변 등록 시:
	> 질문 작성 유저에 답변 등록 메일 발송


3. 관리자 페이지 내용:
    > 상품 분류별 재고 수량 확인
    > 총매출, (브랜드 상품 추첨 위해)남은 매출, 미답변 질문 개수 확인
    > 미답변 질문 목록 보기 + 답변 달기


4. 재고관리 모달창(Ajax를 활용한 비동기 방식):
    > 상품 분류별 확인 가능
    > 검색기능 제공(아이디, 상품명 기준)
    > 각 상품 재고 수량 관리 기능 구현


5. 구매자 현황 페이지:
    > 가입 유저 전체 목록 보기(회원번호, 아이디, 이메일, 주소, 구매금액)
    > 브랜드 상품 당첨 / 미당첨 분류로 목록 보기
    > 모든 고객 추가 정보 서랍식(collapse)으로 보기(구매 목록, 질문 목록)
